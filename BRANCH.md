## What

The application skeleton everything else hangs from:

- **Toolchain**: Vite + Svelte 5 (runes) + TypeScript, pnpm, single root
  `package.json` (no workspace — `pipeline/` scripts run with `tsx` against the
  same dependency tree and import app modules directly; a workspace adds
  ceremony this repo doesn't need).
- **Design tokens**: `app/src/styles/tokens.css` — the entire "Neon Bangkok
  Night" visual system as named custom properties (color, glow, type ramp,
  spacing, motion). Components consume tokens; nothing hard-codes a hex value.
- **Shell**: a ~60-line hand-rolled hash router (`shell/router.svelte.ts`),
  the 4-tab bar (Today · Practice · Progress · Fah), and stub screens for all
  11 routes so every later branch lands features into an existing frame.
- **PWA**: manifest + icons (generated by `pipeline/make-icons.ts` from one
  SVG source of truth), `app/sw.ts` service worker via vite-plugin-pwa in
  `injectManifest` mode, cache names derived from registration scope at
  runtime (required for the multi-environment deploys coming in
  feat/cicd-pipeline).
- **CI seed**: typecheck + unit tests + build on every push/PR, and a minimal
  single-environment deploy of `main` to GitHub Pages (gh-pages branch root).
  feat/cicd-pipeline replaces this with the test→staging→prod promotion.
- **Docs seed**: README skeleton, ADR-001 (stack), ADR-002 (hosting/PWA
  strategy), `docs/retro.md` opened with the planning-phase entry.

## Why these choices

- **Svelte 5 over React/Preact**: the app is animation-heavy (mascot rig,
  glowing tone contours, screen transitions) and Svelte ships motion,
  transitions, and scoped styles in the compiler; runes replace a state
  library outright. Full considered-and-rejected table in ADR-001.
- **Hash routing, no router lib**: 11 flat screens, an installed PWA hides the
  URL bar, `#/` routing sidesteps GitHub Pages' 404-fallback hack entirely,
  and it survives a future Capacitor wrap unchanged.
- **`base: './'` from day one**: relative asset paths are what make
  build-once-promote (one artifact serving /test/, /staging/, and prod) and
  subpath Pages hosting work. Retrofitting absolute paths later would touch
  every asset reference; cheapest decision of the project is making it now.
- **Icons from a script, not an export**: one SVG source of truth rendered to
  all PNG sizes by `pipeline/make-icons.ts` (sharp). Establishes the pattern
  every later asset generator (TTS batch, content packs) follows: generated
  artifacts are reproducible from committed inputs.
- **Fonts as packages**: `@fontsource/noto-sans-thai-looped` (sole learning
  font — looped terminals match how Thai is taught to learners; a pedagogy
  decision, not a taste one) and `@fontsource/kanit` for display headings.
  Self-hosted via the bundle: zero external requests, works offline.

## Definition of done

- `pnpm check && pnpm test && pnpm build` green.
- App deploys to GitHub Pages, installs to an iPhone home screen, and shows
  the tab shell with all 11 stub screens navigable.
- README, ADR-001/002, retro entry 0 committed.
